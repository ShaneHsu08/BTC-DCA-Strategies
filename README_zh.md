一个基于React的Web应用程序，用于比较不同的比特币定投策略。支持标准定投、动态定投和价值平均法三种策略的模拟和比较。

## 关于定投

### 为何定投？
- 看好BTC的长期价值，但希望避免一次性投入带来的择时风险，分散买入时点，降低整体风险。
- 看好BTC的长期价值，但由于资金有限、希望逐步建仓。
- 虽然看不懂 BTC 的短期走势，但认为其长期有持有价值，希望通过定投分散波动风险，以相对平均的低价持有一些 BTC。
- 有‘强制储蓄、长期攒币’的需求，希望通过定投将每月零散资金（如工资结余）转化为 BTC，既避免资金闲置或随意消费，又能长期积累资产。
总之，定投一种资产的前提是你看好这种资产的长期价值。

### 三种策略
1. 标准定投策略：固定金额定期投资，比如每周固定投入500美元。
- 优点：最为简单，多数交易所直接支持，只要在固定时间转入资金执行买入即可。
- 缺点：不管现在价格是高是低都投一样的钱，成本会高一些。
2. 动态定投策略：在标准定投基础上加上一点择时操作，最常见的是使用 RSI 指标（例如 RSI>70 时视为‘市场超买、多数人买入’，此时少买；RSI<30 时视为‘市场超卖、多数人卖出’，此时多买）
- 优点：平均买入价格会低一些，市场下跌时回撤会小一些。
- 缺点：每次定投需根据指标（如 RSI）计算具体投入金额，操作步骤比标准定投多，部分交易所可能不直接支持，需手动计算或借助工具。
3. 价值平均策略（VA）：定期增加固定的持币价值，例如每周目标持币价值固定增加 500 美元，若本周币价猛涨，现有持币价值已超过‘上周持币价值 + 500 美元’的目标，就少买甚至卖掉超出目标的‘超额’部分。
- 优点：可以看作是一种被动的高抛低吸策略，遇到长期震荡市，能更快摊低持币成本。计算也比较简单。
- 缺点：遇到市场暴跌时需要补充大量资金，可能缺乏预算难以执行；遇到市场暴涨时需要卖出大量持币，不利于“囤币党”。我们可以对此作些改进，限制每次投入/卖出的最大金额。

### 何时停止
- 法币本位：比如总投入达到10万美元就停止。
- BTC本位：比如持币数量达到1BTC就停止。
- 策略性止盈：用价值平均策略时，如果市场持续上涨，持仓收益已达到预设目标（或持币成本远低于当前市场价），那就可以考虑止盈离场，开启下一轮定投。
- 遇到紧急资金需求（如医疗、房贷等短期必须支出的费用）时，应暂停或停止定投 —— 定投的前提是‘用闲钱投资’，优先保障日常生活流动性，避免因资金挪用影响生活或被迫亏损离场。
- 若对 BTC 长期价值的核心判断发生改变（如出现根本性监管风险、底层技术被颠覆等），导致‘长期持有’的逻辑失效，应立即停止定投，避免持续亏损。

定投最大的敌人不是市场，而是半途而废的自己。关键是选一个你能长期执行、无压力的策略，然后像吃饭睡觉一样坚持下去。

- **标准定投 (Standard DCA)**: 固定金额的定期投资策略

## 策略说明

### 标准定投 (Standard DCA)
- 每周固定投资相同金额
- 简单易执行，心理压力小
- 适合长期投资者

### 动态定投 (Dynamic DCA)
- 基于RSI指标调整投资金额
- RSI < 自定义阈值: 增加投资（市场超卖）
- RSI > 自定义阈值: 减少投资（市场超买）
- 平衡风险与收益

### 价值平均法 (Value Averaging)
- 目标：投资组合价值每周增长固定金额
- 低于目标时买入，高于目标时卖出
- 设置买卖限制避免极端操作
